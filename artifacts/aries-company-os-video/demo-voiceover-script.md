# Voiceover

Model: gpt-4o-mini-tts
Voice: nova
Format: mp3

## 1. AIRES Company OS / Product demo: Start with the conversation. Leave with governed work.

Here is the Company OS as a concrete product experience. We will start a session, select the exact context it can use, create a document while the conversation is active, and finish by reviewing the synthesis and decisions.

## 2. 01 / Start a session: Begin from Today.

From Today, choose Start session. A session can begin from a live conversation, a customer milestone, a Notion page, a pull request, an imported meeting, or simply a blank inquiry.

## 3. 02 / Define the workspace: Give the session a purpose.

Name the session and its intended outcome, then associate the customer, project, and participants. This creates a bounded workspace for evidence, decisions, artifacts, approvals, and downstream work.

## 4. 03 / Select context: Choose exactly what Cooper can use.

Next, select context. Add the relevant Notion customer page and requirements, then choose the GitHub repository and pull request. Zoom evidence and uploaded files can join the same bundle. Every item stays visible, removable, and source-linked.

## 5. 04 / Session workspace: Context becomes a working briefing.

The session opens with the chosen context already assembled. Cooper gives a concise briefing, surfaces open questions, and keeps the source bundle beside the work canvas instead of burying the team in raw transcripts or repository noise.

## 6. 05 / In-session action: Make a document without leaving the session.

During the session, open Actions and choose Make a document. Select a requirements brief, decision memo, bug brief, or customer follow-up. Here, we create a requirements brief from the selected context and the discussion so far.

## 7. 06 / Review before write: The artifact is drafted. The human still decides.

The draft opens directly on the canvas. Each material claim carries a source anchor. The user can edit scope, challenge an assumption, and preview the exact Notion destination. Publishing remains a separate approval-gated action with read-back verification.

## 8. 07 / End the live work: End the session. Keep the context alive.

When the live work is complete, choose End and synthesize. Aries preserves the artifacts and sources, prepares a recap, and identifies decisions, actions, commitments, and unresolved questions for review.

## 9. 08 / Post-session synthesis: Review the synthesis and decisions.

The post-session view separates synthesis from authority. Confirm the explicit decision, edit the ambiguous one, assign owners to actions, and leave unresolved questions open. Rejected interpretations remain visible in the audit trail instead of quietly becoming work.

## 10. 09 / Governed closeout: One session. Every outcome traceable.

Once approved, the document is read back from Notion, GitHub context stays linked, and the follow-up is ready from the same evidence. The session closes as a durable, traceable account of what happened, what was decided, and what comes next.
