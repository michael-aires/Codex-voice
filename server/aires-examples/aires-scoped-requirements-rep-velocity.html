01

Problem & goal — the why

Problem. A rep faces thousands of leads per project — above human scale — with no trustworthy answer to "what should I do today" or "how do I know I'm done." Because results lag by weeks, their effort is invisible and uncoachable in the moment, and data entry costs time while giving nothing back.

Goal. Make the haystack permanently workable and the day self-directing: a ranked list waiting each morning, a same-day rebate on every input, and every action captured as reportable velocity.

Success metrics

  • Lower time-to-first-meaningful-action after login (rep isn't hunting).
  • Higher daily and weekly activity velocity per rep, and team coverage of the right leads.
  • Lower median response time to hot and aging leads.
  • Higher share of meaningful actions captured as attributable, reportable events.

03

Current state → desired state

Today (the real workflow)

  • Rep opens a generic contact table that shows everything and ranks nothing.
  • They rebuild the same mental or manual filter every morning; prioritize by gut and memory.
  • They act, then maybe log it — entry feels like a tax that only feeds a manager's report.

Desired

  • Dashboard opens to velocity + a ranked "today" list — direction, pre-built.
  • Contact table offers search → filter → saved filters → Smart Lists; the haystack is narrowed for them.
  • Pull mode: when the rep arrives with an intent (a promo, an event, a teammate away), they compose a filter across eight dimensions, save the segment, and run outreach in place — every touch logged.
  • Contact profile makes responding fast and logging near-frictionless; every action becomes a reportable event that moves a visible number the same day.
04

Scope — in / out / non-goals

In scope
  • Search + filter on the contact table
  • Eight composable filter dimensions (attribute, recency, contact gap, event, source, owner, grade, engagement)
  • Saved filters & Smart Lists (per rep / tenant)
  • Rules-based Smart "today" list
  • Bulk / sequenced outreach from a list + consent gate
  • Reportable activity event model + dashboard velocity
  • Per-tenant exclusion defaults
Out of scope (now)
  • ML-based lead scoring (start rules-based)
  • Full marketing journey / automation builder
  • New channels beyond current stack
  • Expanded two-way external sync
  • Configurable list-rules UI (hard-code v1 rules)
Non-goals
  • Not a BI / analytics replacement
  • Not replacing or re-scoping Cooper
  • Not a full manager analytics suite (velocity roll-up only)
  • Not cross-tenant data sharing

05

Data, edge cases & constraints

Data

  • Source of truth: contacts and leads in MongoDB; signals (recency, intent, source) per contact.
  • For the filter dimensions: lead grade, owner, lead source / attribution, event-attendance & presentation-center records, and campaign engagement (opens / clicks via SendGrid).
  • New shape needed: an attributable activity event model (who, what, when, which contact, which rep) — the spine of "reportable."
  • Per-tenant config: exclusion rules, list definitions, project scoping.
  • Scale: thousands of leads per project today; design headroom for 10× in 12 months.

Edge cases & failure modes

  • Empty list (new project) and huge list (perf / pagination) both render usefully.
  • Duplicate or merged contacts don't double-count velocity.
  • Excluded contacts must never reappear in a working list.
  • Concurrent edits and activity writes are idempotent — no double attribution.
  • Pull mode: double-contact suppression across overlapping lists; act-on-behalf attribution + clean hand-back; promo-window lists that go stale.

Constraints & non-functionals

  • Multi-tenant isolation with per-tenant defaults (PATH / Peterson / Creo).
  • Performance: list builds and ranking stay responsive at thousands of leads.
  • Runs on the existing stack — Railway / Cloudflare / Vercel, MongoDB; no AWS.
  • Compliance: bulk outreach respects consent / opt-out (CASL) and SMS A2P 10DLC limits — a filter dimension and a hard send-time gate.

08

Sample acceptance criteria

Expressed given / when / then, so each doubles as a test case (per the definition of done doc).

Smart "today" list
Given a project with 3,000 leads and rules for recency + intent, When a rep opens the dashboard, Then they see a ranked "today" list of ≤ N items, highest-priority first, built in under [target] ms.
Exclusion logic (Creo)
Given a tenant configured to exclude investors and past buyers, When any working list is built, Then no excluded contact appears — verified across saved filters and Smart Lists.
Reportable activity
Given a rep logs a call on a contact profile, When the event is written, Then it is attributed to that rep and contact exactly once, and their dashboard velocity reflects it the same session.
Pull — intent to outreach
Given a rep composes "opened the two-bedroom promo" + grade ∈ {A, B} and saves it, When they launch bulk outreach, Then opted-out and recently-contacted contacts are suppressed, the rest are messaged, and every touch is logged and attributed.
