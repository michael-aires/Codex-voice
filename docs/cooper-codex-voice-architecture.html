Use the simplest hands.

Opening Safari is not a Computer Use problem. It is a deterministic tool call. Reserve visual reasoning for workflows that truly require perception and adaptive UI interaction.

Deterministic action

Fast, explainable, and reliable for known targets.

Best for: opening apps and URLs, arranging known workspaces, running a trusted Shortcut, or calling a stable API.

Model-driven Computer Use

Flexible and situational for interfaces without a better control surface.

PerceiveCapture screenshot + state
PlanReceive typed UI actions
ActClick, type, scroll
VerifyReturn a fresh screenshot
Best for: complex or changing visual interfaces where APIs, structured browser control, and deterministic tools are unavailable.
The routing order is: direct API or connector → deterministic local tool → structured browser automation → Computer Use.

Control Codex through a supported programmatic surface.

Do not make Cooper click around inside Codex Desktop. Use a local adapter so task IDs, events, approvals, and cancellation remain first-class product state.

App-server is the best fit for Cooper’s local UI

It exposes the exact primitives Cooper needs: create, resume, fork, list, name, steer, interrupt, and archive threads; start turns; stream item lifecycle; and answer command or file-change approvals.

Use it forPersistent tasks with live progress, approval cards, diffs, and a full activity timeline.
Design responseWrap the protocol in a versioned adapter and pin the Codex CLI version because app-server is documented as experimental.
Recommended primary path. Keep codex exec --json as a simpler fallback for one-shot jobs and bridge recovery.

Cooper can propose work during the conversation or after the call. In both cases, execution begins from the same typed proposal and produces the same event trail.

TIER 0

Pre-authorized open actions

Named routines may open allow-listed apps and URLs immediately. No typing, sending, scripts, or settings changes.

TIER 1

Approve task start

Start a scoped Codex turn or supervised Computer Use run. Later Codex permission prompts still surface separately.

TIER 2

Approve consequential action

External communication, login, purchase, deletion, commit, push, production write, expanded permissions, or sensitive data use.

Build the narrow vertical slice first.

The highest-value first release is voice → visible proposal → approval → real Codex thread → live canvas events. The morning routine is a small deterministic feature in the same release.

Phase 1 · 3–5 days

Codex bridge spike

Prove the protocol with one repository and one running task.

  • Process manager + health check
  • thread/start and turn/start
  • Event normalization
  • Interrupt and reconnect
Phase 2 · 4–6 days

Approvals + canvas

Connect real Codex state to the UI Cooper already has.

  • Task proposal schema
  • Command/file approvals
  • Diff, test, and result views
  • Post-call dispatch
Phase 3 · 2–3 days

Personal routines

Turn “I’m ready to work” into a safe, immediate launch.

  • Named routine config
  • App + URL allow-list
  • Parallel deterministic launch
  • Visible success/failure receipt
Phase 4 · 1–2 weeks

Computer Use loop

Add perception only after the deterministic routes are solid.

  • Isolated harness
  • Screenshot/action cycle
  • Domain + app policy
  • Replay and injection defenses
