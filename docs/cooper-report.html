Capabilities & Hardening Report

Cooper — Realtime Voice Agent

A local-first React 19 + Express progressive web app: an AIRES executive voice assistant that joins live meetings over the OpenAI Realtime v2 API (WebRTC) and turns each call into polished Markdown and HTML artifacts via the OpenAI Responses API.

Single user · one shared password Server-relay key never reaches the client Silent by default · wake-word activated Stack: React 19 · Vite 6 · Express 4.21 Grounded in server.js (1110 LOC) · main.jsx (1707 LOC)
01

Overview

Cooper is a single-Node-process system for one user ("Michael", CTO/CPO). The browser PWA is the only client; the Express server is the only backend, doubling as the SPA host (Vite middleware in dev, static dist/ in prod) and as a credential-shielding relay to OpenAI. It listens silently during meetings, speaks only when called on or woken by a wake word, then generates artifacts from a fixed set of prompt-chain recipes. All durable state lives in one local JSON file; artifacts are written to local disk.

Live meeting audio

OpenAI Realtime v2 over WebRTC. Mic capture via getUserMedia, remote voice cedar, server-VAD turn detection, far-field noise reduction.

Silent by default

create_response:false means Cooper never auto-replies to VAD turns. It speaks only on "Call Cooper", a typed prompt, or the wake-word regex.

Artifact recipes

6 fixed multi-step chains: post-call kit, execution plan, follow-up, code sketch, PRD, and a standalone HTML prototype — Markdown or HTML output.

Live transcript & wake word

Per-turn transcript capture with gpt-4o-mini-transcribe; a regex on completed turns triggers a response when "cooper" is heard.

Safe artifact rendering

Markdown via markdown-it + DOMPurify; Mermaid lazy-rendered; HTML prototypes in a sandboxed <iframe> without allow-same-origin.

Crash-recoverable jobs

Single in-process worker with pacing, retry/backoff, and boot-time recovery of stuck running jobs back to queued.

Installable PWA

Web manifest + cache-first service worker + app icon. Runs and installs as a standalone progressive web app.

Live updates over SSE

Every datastore write broadcasts state.updated over Server-Sent Events; clients refetch state and fire completion notifications.

Calendar tool (client stub)

check_calendar(date,time) is registered as a Realtime tool but executes locally against a hardcoded sample calendar — no real backend.

How it works

Two distinct paths. The call path establishes a WebRTC session with OpenAI Realtime through a server SDP relay and streams live audio + events. The post-call job loop turns an ended call's transcript into a generated artifact through a single, paced, retry-aware in-process worker.

Post-call job loop — artifact generation

Pacing: lastGenerationAt + jobDelayMs (15s default) between model calls. Retry: 408/409/429/5xx → RetryableJobError honoring Retry-After, re-queued up to maxAttempts (3); else failed with a manual retry button. Boot recovery resets stuck running jobs to queued (server.js:1063-1090). HTML output is extracted from the final draft, else a fallback document is used.

Persistence

One JSON file data/cooper.json {calls, artifacts, jobs}, written through a serialized in-process writeQueue — no cross-process locking; whole-file rewrite per change.

Models

Realtime gpt-realtime-2 (effort low); work model gpt-5.4 (effort medium, max_output_tokens 6500) with attempt-indexed fallback.

Auth

Constant-time password compare → HMAC-SHA256 signed cookie cooper_session (HttpOnly · SameSite=Lax · TTL Max-Age).

03

What's good

Cooper gets the high-leverage decisions right. These instincts should be preserved through any refactor.

✓

Server-side key relay

The server relays both the Realtime mint and the Responses API with a server-side Bearer header. OPENAI_API_KEY never reaches the browser. server.js:200-243, 658-703

✓

Constant-time auth compare

safeCompare length-checks then timingSafeEquals the password — no timing oracle on login. server.js:984-991

✓

Signed HttpOnly cookies

Sessions are HMAC-SHA256 signed and verified before the expiry check; the cookie is HttpOnly + SameSite=Lax + scoped Path + TTL Max-Age. server.js:954-982, 1012-1020

✓

Iframe sandbox without allow-same-origin

HTML prototypes render in a sandboxed iframe that deliberately omits allow-same-origin — generated scripts cannot touch the app origin, cookies, or storage. main.jsx:1442

✓

DOMPurify-sanitized Markdown

markdown-it output passes through DOMPurify.sanitize before injection into the DOM. main.jsx:1588

✓

Silent-by-default UX

create_response:false in baseSession means the assistant never auto-responds to ambient VAD turns — it speaks only when called on. server.js:116-141

✓

Crash-recoverable job loop

On boot, any job stuck in running is reset to queued with a recovered log, and the worker is re-kicked. server.js:1063-1091

✓

Write serialization

All datastore writes are serialized through the writeQueue promise chain, avoiding intra-process write interleaving. server.js:781-791

✓

Artifact path-traversal mitigated

artifactFileName takes the basename via .pop(), stripping directory components before any disk read. server.js:873-878

✓

Installable PWA

Web manifest + cache-first service worker + icon make Cooper installable and resilient to flaky connectivity for shell assets. public/manifest.webmanifest · public/sw.js

04

Where it needs hardening

Severities are calibrated to what Cooper actually is — a single-user, local-first tool, not a public multi-tenant SaaS. No critical or remotely-exploitable-without-credentials vulnerability was found. Residual risk concentrates in authentication hardening and data-at-rest gaps.

High Medium Medium/Low (annoyance) Low

No login rate limiting

High

The single shared password is the entire perimeter and the login handler has no counter, delay, or lockout. Anyone who can reach the port can attempt unlimited guesses — strength collapses to password entropy × guess rate.

POST /api/auth/login · server.js:40-61
Fix: Add per-IP failed-attempt throttling with exponential backoff and temporary lockout; add a global attempt budget as a backstop; enforce a minimum password length/entropy at startup.

Session secret defaults to the app password

High → Med

When COOPER_SESSION_SECRET is unset, the HMAC signing key is the password. No key separation: a password leak becomes an offline session-forgery capability, and rotating the password silently invalidates every session.

server.js:23 · used by signPayload 980-982
Fix: Require an explicit, independent, high-entropy COOPER_SESSION_SECRET. Fail closed at startup (or warn loudly) if it falls back to the password.

Secure cookie flag only in production

Medium

The cookie carries HttpOnly + SameSite=Lax always, but Secure is gated on NODE_ENV==="production". In the default dev posture the cookie is transmittable over plain HTTP and can be sniffed if exposed over non-TLS, non-localhost transport.

secure:isProduction · server.js:56, 67, 1017
Fix: Set Secure based on actual HTTPS (req.secure / X-Forwarded-Proto) rather than NODE_ENV; document HTTPS-only for any non-localhost exposure.

No CSRF protection on mutating routes

Medium

Auth is by ambient cookie with no CSRF token, required custom header, or Origin/Referer check on state-changing POST/PATCH routes. SameSite=Lax blocks the classic form-POST and image-tag vectors, so residual exposure is narrow — hence Medium.

server.js:280-422 (calls / transcript / end / artifacts / retry)
Fix: Require a custom header (forcing a preflight) or validate Origin/Referer against an allowlist on mutating routes; or use a double-submit CSRF token.

Prompt injection via transcript + custom prompt

Medium

buildWorkPrompt concatenates the full untrusted transcript and Michael's free-text custom prompt into the Responses prompt. Crafted utterances could steer artifact content. Bounded by the iframe sandbox and no tool execution on the Responses path — harm is misleading content, not code execution.

server.js:705-745 · render main.jsx:1442, 1588
Fix: Delimit and label untrusted transcript / custom-prompt blocks as data, not instructions; keep the sandbox and DOMPurify; continue treating all model output as untrusted at the render boundary.

Plaintext meeting data at rest

Medium

Every transcript turn and generated artifact is persisted unencrypted under data/ with no retention or expiry controls — data accumulates indefinitely. For a local executive assistant this is the central confidentiality concern; the threat requires host/filesystem access.

data/cooper.json (updateDb 781-791) · data/artifacts/ (completeArtifact 613-656)
Fix: Encrypt sensitive fields/files at rest (or formally require host full-disk encryption), add a retention/purge mechanism, and document the on-disk data footprint.

html_prototype iframe allows scripts/popups/modals

Med → Low

The prototype iframe grants allow-scripts + allow-popups + allow-modals. Generated HTML can script, open popups, and show modals — an annoyance / redirect surface. Origin-isolated (no allow-same-origin), so cookies and the app origin stay safe.

main.jsx:1442
Fix: Tighten the sandbox tokens — drop allow-popups / allow-modals unless required; consider a sandbox CSP.

Single JSON file — durability & scale

Low (Reliability)

All state is one JSON file with whole-file rewrites and no cross-process locking, no durability or backups. It works for a single user but offers no concurrency story and no recovery beyond the running → queued job reset.

readDbRaw 770-779 · writeQueue 781-791
Fix: Migrate to an embedded transactional store (e.g. SQLite) for atomic writes and crash safety; add periodic backups and a documented retention policy.

Single-process worker — no horizontal scale

Low (Reliability)

One in-process job worker means no horizontal scale; a process death mid-write can lose a job (partially mitigated by boot recovery). Artifacts sit on local disk with no object storage.

processQueue 479-514 · runJob 516-611
Fix: Externalize the queue and artifact storage if scale is ever needed; until then, accept for single-user and rely on the existing recovery path.

No observability

Low

No request logging, metrics, tracing, or health endpoint; only verbose console errors. There is no audit trail (single shared credential), and SSE broadcasts to all connected clients with no per-session scoping.

server-wide · GET /api/events 250-263
Fix: Add structured request logging and a /healthz; scrub internal detail from client-facing errors; scope SSE per-session if multi-user is ever introduced.

No automated tests

Low → High (debt)

There are no tests anywhere in the repo, no CI, and no linter config. The auth, retry/backoff, prompt-assembly, and crash-recovery logic — exactly the parts where regressions are most dangerous — are entirely unguarded.

repo-wide (no test/ · no CI workflow)
Fix: Add unit tests for safeCompare / session signing, the retry classifier, buildWorkPrompt, and crash recovery; add a CI gate (lint + test) on every push.

06

Roadmap

Phased hardening, ordered by leverage. Phase 1 is cheap and high-value; Phases 2–3 are defense-in-depth and durability investments to revisit if Cooper grows beyond a single local user.

Phase 1 — Must fix

Close the auth gaps

  • Add per-IP login throttling + backoff + lockout (closes the brute-force path).
  • Require an explicit, independent COOPER_SESSION_SECRET; fail closed on fallback.
  • Set the Secure cookie flag on actual HTTPS, not NODE_ENV.
  • Add a minimal test suite + CI gate for auth, retry classification, and crash recovery.
Phase 2 — Should fix

Defense in depth & data hygiene

  • Origin/Referer (or custom-header) check on all mutating routes.
  • Encrypt meeting data at rest (or require host full-disk encryption) + add retention/purge.
  • Delimit untrusted transcript / custom-prompt blocks in buildWorkPrompt.
  • Add structured request logging, a /healthz, and scrubbed client-facing errors.
Phase 3 — Scale & polish

Durability & growth readiness

  • Migrate the JSON store to an embedded transactional DB (e.g. SQLite); add backups.
  • Tighten the prototype iframe sandbox tokens; pin the DOMPurify config.
  • If multi-user is ever needed: per-user accounts, audit logging, per-session SSE scoping.
  • Externalize the queue + artifact storage to enable horizontal scale.
